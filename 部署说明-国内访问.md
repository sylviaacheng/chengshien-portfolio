# 网站部署说明

这个项目已经整理成可直接部署到 `GitHub + Vercel` 的版本，并额外处理了几个会影响中国大陆访问的问题：

- 去掉了 `Google Fonts` 外链，改成系统字体回退
- 去掉了 `Unsplash` 外链图片，改成本地图片
- 修复了生产环境下 `/src/assets/...` 图片路径失效的问题
- 新增了 `vercel.json`

## 先说结论

你现在这套方案可以这样走：

1. 代码放到 GitHub
2. 在 Vercel 导入 GitHub 仓库自动部署
3. 在 Vercel 里绑定你的阿里云域名
4. 在阿里云解析里按 Vercel 页面提示添加解析记录

这套方式会比 `*.vercel.app` 域名更适合中国大陆访问，但**不能保证 100% 稳定直连中国大陆**。如果你后面要追求更稳定的国内打开速度，建议做一套阿里云国内镜像站或直接放到阿里云中国大陆节点。

## GitHub 步骤

由于当前环境里没有可用的 `git` 命令，你可以用下面两种方式二选一：

### 方式一：本机安装 Git 后执行

```bash
git init
git add .
git commit -m "init portfolio site"
git branch -M main
git remote add origin https://github.com/你的用户名/chengshien-portfolio.git
git push -u origin main
```

### 方式二：直接在 GitHub 网页上传

1. 登录 GitHub
2. 新建仓库：`chengshien-portfolio`
3. 点击 `uploading an existing file`
4. 把整个项目文件拖进去上传

## Vercel 步骤

1. 登录 Vercel
2. 点击 `Add New` → `Project`
3. 选择刚才的 GitHub 仓库
4. 框架会自动识别为 `Vite`
5. 确认构建配置：
   - Build Command: `npm run build`
   - Output Directory: `dist`
6. 点击部署

## 阿里云域名绑定

推荐优先绑定：

- `www.你的域名.com`

然后再补：

- `你的域名.com`

在 Vercel 项目里进入 `Settings` → `Domains`，添加域名后，Vercel 会给出**当前项目对应的准确 DNS 记录**。你只需要回到阿里云域名解析后台，照着 Vercel 提示填写即可。

常见情况：

- `www` 子域名：通常使用 `CNAME`
- 根域名：通常使用 `A` 记录

## 中国大陆访问建议

为了尽量提高国内可访问性，建议这样做：

1. 不使用 `*.vercel.app` 对外访问
2. 一定绑定你自己的阿里云域名
3. 页面尽量使用本地图片、本地字体、少依赖国外第三方脚本
4. 如果后面要长期面向中国大陆用户，尽快准备备案并考虑阿里云国内部署
